# AMBUSH Direction-Locked Design Pack

One claim underwrites the whole company: that *filtered, consensus-gated fan-out of heterogeneous agents produces more trustworthy security findings than one strong agent*. Everything downstream — the war-room UX, the Pro tier, the unit economics — is a way of producing, rendering, or selling that filtered signal. So the three lanes below are not parallel workstreams; they are one artifact at three altitudes. The eval **defines** the structured Finding schema, the consensus key, and the evidence gate. The UX **renders** exactly those objects (clusters, diversity, quarantine, signed receipts). The GTM **monetizes** the boundary between producing trust (paid) and verifying it (free, forever). The spine that runs through all three is the same line: *you pay to generate a validated, signed artifact; anyone verifies it for free.*

## Validation Experiment — the kill/greenlight gate

Before a single product ticket, a 3-week offline eval (1 pilot week + 2 full) run by 1–2 AppSec engineers answers GO/KILL. It lives in a new `eval/` sibling that does not touch product code, and its output is one `scorecard.csv` plus an adjudicated, labeled corpus that *becomes* the product's `Consolidate` spec. Today there is no structured finding output — lanes write freeform `findings/<id>.md` and `consolidate()` merely concatenates them into `RUNBOOK.md` (`swarm-orchestrator.ts:286`). The eval's first deliverable is therefore the schema itself: a per-lane `findings/<id>.jsonl` carrying `cwe`, `location`, typed machine-checkable `evidence[]` (poc / taint_path / test / sast_corro / citation), a family-tagged `model`, and `lane_confidence`. The **consensus key** `K = (normalized_file_path, CWE_class, sink_or_symbol)` with ±10-line overlap is the load-bearing definition of "two lanes corroborate."

The eval is deliberately adversarial toward our own thesis. It tests two hypotheses across seven arms run on one paired task set:

- **H1 (signal):** the full pipeline — A4 = N-lane union → consensus (support ≥ k) → evidence/oracle gate — must beat both A0 (one strong agent) and A1 (strong agent + Semgrep) on finding-level precision without shedding fan-out's recall.
- **H2 (independence):** cross-family panels must add real signal over same-family panels — because the #1 existential risk is that BYO agents are the same few frontier models, so 3 lanes "agreeing" can be one model hallucinating 3 times and consensus *promoting* the slop.

H2 is measured directly. For each task, run identical pipelines on a **HOMO-6** panel (six lanes of one family, varied only by seed/persona) and a **HETERO-6** panel (≥3 families: Claude, GPT, Gemini, plus one OSS like Qwen-Coder). Compute pairwise error-vector correlation, derive **effective independent voters** `N_eff = N / (1 + (N−1)ρ̄)`, and measure the **correlated-FP promotion rate** — the fraction of *false* clusters that nonetheless reach support ≥ k. Literature suggests family-correlated panels can collapse to ~2.5–3.6 effective votes, so this is not paranoia.

Corpora span four tiers, and **Tier D (negative controls) is non-negotiable** — without code whose correct answer is "no finding," you can only measure recall, never precision or the slop-filter. Tier A: execution-oracle real CVEs (CyberGym ~80 tasks, CVE-Bench ~20) where a PoC that crashes pre-patch and runs clean post-patch is the strongest possible oracle. Tier B: commit-level labels (PrimeVul, whose *paired safe function* is a built-in negative). Tier C: planted whole apps (Juice Shop, NodeGoat, OWASP Benchmark with deliberate TP *and* FP cases). Tier D: PrimeVul-safe halves, OWASP-Benchmark FPs, and a frozen hardened library release. Because these benchmarks likely leak into training data, a **private 20–30 finding planted set** (hand-injected CWE-89/79/22/78/502 into recent clean OSS, with matched clean originals) is the leak-proof tie-breaker; report public-vs-private precision deltas. The wedge framing is primary: give a lane the repo at the *vulnerable commit* and ask "what's exploitable?"; ground truth is the fixing commit. CTF/exploit tasks are the secondary funnel slice, not the headline number.

Decisions are made on bootstrap CIs and paired McNemar tests, never point estimates, with two-annotator blind adjudication (target κ ≥ 0.7).

**GREENLIGHT (all four must hold):**
1. **Signal:** `P_{A4} − max(P_{A0}, P_{A1}) ≥ +0.15` absolute, non-overlapping CIs, McNemar p < 0.05.
2. **Recall retention:** `R_{A4} ≥ 0.90 × R_union` — the filter keeps ≥90% of fan-out's recall.
3. **Independence pays:** HETERO `N_eff ≥ 1.5 × HOMO N_eff` and HETERO precision-at-iso-recall beats HOMO.
4. **Slop contained & cheap:** correlated-FP promotion (HETERO, A4) < 0.10, and cost-per-validated-finding ≤ 3× A1.

**KILL/PIVOT (any one):** A4 precision CI overlaps A1 at iso-recall (the cheaper baseline wins); HETERO ≈ HOMO (independence is fictional, corroboration is theater); HOMO consensus promotes correlated FPs such that `P_{A4} < P_{A0}` (consensus is *actively harmful*); or recall retention < 0.75 (the filter is just an expensive single agent). A **YELLOW** — signal real but only with forced family diversity — still ships, but reshapes the roadmap so diversity enforcement and the slop-filter become the entire pitch and sandboxing/attestation demote to secondary.

Budget is dominated by hetero API calls: ~250 tasks × 6 lanes × ~$0.30–1.50 ≈ **$1.5–4k**, cached aggressively so arms A2–A6 are pure post-processing. One full run also emits the end-to-end attestation (in-toto statements, Merkle root via the engine's real `swarm-crypto`, a prototype `ambush verify`) so the demo's verifier is proven real, not vapor — a side-quest, not part of GO/KILL.

## Operator UX — the war room

The eval's objects need a home, and today there isn't one: the app is tab-routed, showing exactly one of `SwarmView | IntelPane | ReceiptsPane`. The core IA move is to **collapse the three tabs into one persistent three-pane war room** where the swarm tree, the focused lane, and the signed audit stream are visible *simultaneously*, under an always-on scope/authorization banner with a kill-switch.

The **left OperationTree** (extending the existing `w-80` aside + `VectorCard`) groups Vectors by role/lease, each row showing a status dot, codename, a **lease chip**, and ALLOW/DENY tallies plus a findings count `⚑N`. The lease chip is the visible promise that *worktrees are not the boundary*: `ro` (read-only worktree), `rw▣` (sandboxed write in a container/microVM), `net⊘` (egress denied), `net⊕scope` (scoped allowlist). When an operator points 24 BYO Vectors at untrusted code, `rw▣ net⊘` is the posture they can *see*. The **center CenterStage** is a segmented mode switch — Lane (the reused `TerminalPane` + a per-lane `IntelGraph` rail) / Findings / Runbook — so intel and runbook become modes, not peer tabs. The **right AuditStream** refactors the polled `ReceiptsPane` table into a live, newest-on-top signed-receipt feed subscribed to a new streaming `evt:receipt` event, with a whole-chain verify badge in its footer.

Three **signature moments** carry the demo, and each maps one-to-one onto an eval object:

1. **Real-time DENY.** An out-of-scope `curl` triggers an `evt:receipt` with `verdict==='DENY'`; the AuditStream row goes full-red and sticky while a `DenyToast` fires bottom-right: `⛔ DENIED in real time · net.connect → pastebin.com:443 · policy: scope.egress · receipt #312 · nothing left the box ✓`. This requires the highest-leverage net-new work — the streaming event plus `ReceiptSummary.args/reason` fields, since today receipts carry no arguments and an argument-level deny *cannot even be rendered*. Toast and row are driven by the same event, so they can never disagree.

2. **Slop quarantine** inside `FindingsReview` — the painkiller made visible. Two columns, **Corroborated** vs **Quarantine**, where the eval's correlated-error guard becomes literal UI: a **model-diversity meter** (`●●○`) reads *distinct frontier models*, not lane count, so "3 lanes / 1 model" still shows `●○○` and stays quarantined. A cluster lands in Quarantine iff `distinctModels < 2` OR no lane carries reproducible evidence. Each `FindingCard` shows severity, `file:line`, CWE, corroboration (`lanes N / models M`), and a checkable evidence row — `PTY` (deep-link to terminal scrollback), `PoC`, `receipt#` — where a finding with no allowed-receipt evidence *cannot be confirmed*.

3. **Export Attestation + Verify.** A two-step `AttestationDialog` bundles confirmed findings ∪ their evidence receipts ∪ the full hash-chained receipt log ∪ in-toto statement ∪ policy-pack hashes, Ed25519-signed by the local operator key, and the reused `VerifyBadge` re-derives the Merkle root on a clean machine.

**Build delta:** keep `TerminalPane` and the terminal hub verbatim; extend `VectorCard`, `StatusBar` (add trust-kernel daemon dot + chain head), and `OperationSetup` (one-click Security-review / Dependency-vet / Diff-review templates that pre-fill scope + egress allowlist + lease defaults); replace `App.tsx` tab routing with a `<WarRoom/>` shell and refactor `ReceiptsPane → AuditStream`. Twelve new components, but the four demo beats only require the `WarRoom` shell to land first, then ~3 small components per beat. New shared types — `VectorLease`, `Finding`, `FindingCluster` with `distinctModels`, `ScopeAuth`, `Attestation` — and IPC channels (`evtReceipt`, `evtFinding`, `evtCluster`, `attestationExport/Verify`) are the contract the React dev builds against.

## Packaging & GTM — the Burp playbook, numeric edition

The free/paid line follows one principle: **free = the swarm + the safety floor + the verifier (commodity, trust, funnel); Pro = making the swarm's output trustworthy and exportable (the painkiller); Org = making that trust shared, identity-bound, and retained.**

**OSS Core (MIT, free forever):** fan-out up to 100 lanes, worktree isolation, PTY terminals, any-CLI-agent profiles, the intel vault, *basic* concatenation Consolidate, the fail-closed trust-kernel governance floor, and crucially **`ambush verify`**. A security tool that paywalls the safety floor never gets pointed at untrusted code, and an attestation you must pay to check is worthless — so the verifier is free on both GTM and license-posture grounds.

**Pro (~$400/seat/yr):** the validated slop-filter with cross-model-family decorrelation, the container/microVM sandbox, synthesis-grade Consolidate, signed Attestation *export*, and curated policy packs. The clean boundary is the **sign/verify split**: you pay to *generate* a validated, synthesized, signed artifact; anyone verifies it free. $400 sits $75 under Burp Suite Pro's $475/seat in the expense-it-no-procurement zone, inside the Semgrep ($35/contributor/mo) and Snyk ($25/dev/mo) bands, and well under ZeroPath per-operator. Position literally as *"Burp for the agent swarm."*

**Org/Audit (~$15–40k/yr):** shared audit server, SSO/SAML + RBAC, retention/legal-hold, and a receipt SDK for CI/CD and deal-room embedding. The trigger is structural, not feature-greed — the moment provenance must be shared, identity-bound, and retained across a team and handed to external parties. The exact conversion event to instrument: **the first time a Pro user's attestation is independently `ambush verify`'d by an external counterparty** (acquirer's board, deal counsel, an auditor) who says "we want all of them this way."

**Unit economics make the slop-filter a scheduler, not a report-cleaner.** A heterogeneous review lane costs ~$6 blended. The naive operator, distrusting any single lane, drags the slider to 100 → ~$600/burst of correlated frontier-model noise. The governed path caps lanes per family, stops at corroboration saturation, and kills quarantined slop lanes early → ~18 diverse lanes (~$108) + per-*finding* cross-family adjudication (~40 × $0.20 = $8) ≈ **$116/burst, ~80% fewer lanes.** The discipline is an architectural law: cost is committed at *spawn*, so the filter must cap lanes up front and **adjudicate findings, never re-run lanes** (per-lane re-validation would re-add $108 and destroy margin). For a boutique running 25 engagements × 2 bursts/yr that's $30k → $5.8k token burn — **~$24k saved on a $400 seat, ~60× return**, with Pro at ~95% gross margin (BYO keys, Ambush COGS ≈ $0). For the M&A-DD lead it replaces a $25–50k boutique engagement at <2% of cost while producing a board-grade signed artifact — *nothing left the box.* The license posture stays clean throughout: MIT control plane (proprietary Pro modules permitted), Apache-2.0 engine (linkable, patent grant), GPL OpenKnowledge invoked subprocess-only — never vendored.

## Build order for the first 8 weeks

The eval gates everything; the demo is the close. Sequence accordingly.

- **Weeks 1–3 — Eval first, in parallel with kernel extraction.** Stand up `eval/`, clone corpora, define the Finding schema + consensus key + evidence gate, run the 15-task pilot, then the full A0–A6 run with HOMO-6 vs HETERO-6 panels. End Week 3 with the GO/KILL scorecard. **No renderer work depends on the verdict, but the schema does** — it is the single shared artifact handed to the UX and product tracks. Concurrently, begin extracting the engine's swarm-spine/crypto/policy/consensus into a **local trust-kernel daemon**, and delete the absent Chio shim. The eval's attestation side-quest (Merkle + Ed25519 + `ambush verify` prototype) doubles as the kernel's first integration test.

- **Weeks 3–5 — Fail-closed governance.** With the kernel daemon running, implement argument-level fail-closed policy for shell/exec/network/filesystem, plus the container/microVM sandbox for any `rw▣` write/exploit lane (worktrees are not a boundary). Wire the streaming `evt:receipt` with `args`/`reason` — the prerequisite for the demo's real-time DENY, and the thing the current `ReceiptSummary` cannot express. Ship the host-allowlist network policy that denies the out-of-scope curl.

- **Weeks 4–6 — Demo UX.** Land the `WarRoom` three-pane shell first (one-file refactor of `App.tsx`/`SwarmView.tsx`) so the four beats have a home, then build them in order of leverage: AuditStream + DenyToast (beat 2, real-time DENY), FindingsReview + model-diversity meter fed by the consensus output of the *now-greenlit* pipeline (beat 3, quarantine), lease chips on VectorCard (beat 1, posture), and AttestationDialog + VerifyBadge bound to the kernel crypto (beat 4, signed export). The synthesis runs inside the existing `consolidate()` path, now emitting clusters with `distinctModels`.

- **Weeks 6–8 — Design-partner outreach.** Recruit 5–10 boutique AppSec/pentest shops + 1–2 M&A-DD leads; give Pro free for 90 days in exchange for ≥2 real engagements and a named case study. The killer demo *is* the conversion event — 24 heterogeneous Vectors on untrusted code, an out-of-scope curl denied in real time, 3 lanes corroborating while a lone mono-model finding is quarantined, and an Ed25519 bundle that verifies on a clean machine. Instrument for the first externally-verified attestation: it is the metric that predicts every Org upsell.

If the eval kills H1 or H2, stop at Week 3 — the cheapest possible failure, before a line of UX is built on a false premise.
